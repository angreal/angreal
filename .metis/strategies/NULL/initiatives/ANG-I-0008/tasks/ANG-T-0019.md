# Verify all documentation code examples against implementation

## Objective

Systematically verify every code example in the angreal documentation against the actual implementation. Each document must be checked to ensure import paths, function signatures, parameter names, return types, and usage patterns match the current codebase.

## Verification Instructions

For each document:

1. **Read the documentation file** - Identify all code examples
2. **Find corresponding implementation** - Locate the Rust source in `crates/angreal/src/` or Python bindings
3. **Find corresponding tests** - Check `py_tests/` for test files that exercise the documented API
4. **Verify each code example**:
   - Import paths are correct (e.g., `from angreal.integrations.git import Git`)
   - Function/method signatures match (parameter names, types, defaults)
   - Return types/formats match (e.g., tuple vs object)
   - Property vs method distinction is correct
   - Example code would actually work if copied
5. **Mark checkbox when verified** - Add notes if issues found
6. **Fix any issues** - Update documentation OR file a bug if implementation is wrong
